# Contributing

Thanks for your interest in contributing! 🎉
Contributions are welcome, but please read this guide before getting started.

## ⚠️ Important Note on Contributions

While contributions are appreciated, please understand:

* This project is maintained in **limited spare time**
* There is **no guarantee** that contributions will be reviewed, accepted, or merged quickly
* Some contributions may **not be incorporated at all**, depending on scope and priorities

If you're okay with that—great, you're welcome to contribute 👍

---

## 🧭 Ways to Contribute

You can contribute in several ways:

* Reporting bugs
* Suggesting improvements
* Improving documentation
* Submitting code changes

---

## 🐛 Reporting Issues

Before opening a new issue:

* Check if the issue already exists
* Provide clear steps to reproduce
* Include relevant logs or screenshots if applicable

---

## 💡 Suggesting Features

Feature requests are welcome, but keep in mind:

* Not all features will be implemented
* Larger ideas may need discussion first

Please describe:

* The problem you're solving
* Your proposed solution
* Any alternatives considered

---

## 🔧 Submitting Changes

1. Fork the repository
2. Create a new branch:

   ```
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Keep commits clean and meaningful
5. Push your branch and open a Pull Request

---

## 📋 Pull Request Guidelines

* Clearly describe what your PR does
* Keep changes focused and minimal
* Link related issues if applicable
* Be open to feedback or requested changes

---

## ⏳ Review Process

* Reviews happen **when time allows**
* There may be delays or no response for a period of time
* If a PR becomes stale, it may be closed

---

## 🚫 What to Expect

* Not all contributions will be merged
* Design or architectural decisions are at the maintainer’s discretion
* The project direction may change over time

---

## 🙏 Thanks

Your time and effort are appreciated—even if your contribution isn't ultimately merged.
